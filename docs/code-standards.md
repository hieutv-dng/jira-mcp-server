# mcp-jira-tools: Code Standards & Conventions

## Overview

Tài liệu này định nghĩa coding conventions, patterns, naming standards áp dụng cho mcp-jira-tools project.

**Language:** TypeScript (ES2022)
**Module System:** ES modules (NodeNext)
**Code Style:** Minimal — prioritize clarity over style enforcement

## File & Naming Conventions

### File Naming

| Type | Pattern | Example | Notes |
|---|---|---|---|
| Source files | kebab-case.ts | `jira-client.ts` | Not used; follow category |
| Organized by folder | category/file.ts | `jira/client.ts` | Folder = feature, file = implementation |
| Index files | index.ts | `src/index.ts`, `src/shared/index.ts` | Entry points, re-exports |
| Utilities | utils.ts | `shared/utils.ts` | General utilities |
| Config files | config.json, .env | mcp-config.json, .env.example | No ext change |

**Folder Structure:**
- `src/` — Source code (TypeScript)
- `src/jira/` — Jira-specific: client, tools, formatter
- `src/shared/` — Shared utilities, types
- `dist/` — Compiled output (generated by `npm run build`)

## Class & Interface Patterns

### JiraClient Pattern (Singleton)

```typescript
export class JiraClient {
  private baseUrl: string;
  private pat: string;
  private client: AxiosInstance;

  constructor(baseUrl: string, pat: string) {
    this.baseUrl = baseUrl;
    this.pat = pat;
    this.client = this.initializeClient();
  }

  private initializeClient(): AxiosInstance {
    const instance = axios.create({
      baseURL: this.baseUrl,
      timeout: 15000,
      headers: { 'Authorization': `Bearer ${this.pat}` }
    });
    instance.interceptors.response.use(
      res => res,
      err => { throw this.formatError(err); }
    );
    return instance;
  }

  // Public methods
  async searchIssues(jql: string): Promise<SearchResult> { }
  async getIssue(key: string): Promise<Issue> { }

  // Private helpers
  private formatError(err: unknown): Error { }
}

// Singleton export
export const jiraClient = new JiraClient(
  process.env.JIRA_BASE_URL ?? '',
  process.env.JIRA_PAT ?? ''
);
```

## Error Handling

### Error Wrapper Pattern

```typescript
// shared/utils.ts
export function formatToolError(error: unknown): {
  code: string;
  message: string;
} {
  if (error instanceof Error) {
    if (error.message.includes('timeout')) {
      return { code: 'API_TIMEOUT', message: error.message };
    }
    if (error.message.includes('401')) {
      return { code: 'AUTHENTICATION_FAILED', message: 'Invalid Jira PAT token' };
    }
    return { code: 'INTERNAL_ERROR', message: error.message };
  }
  return { code: 'UNKNOWN_ERROR', message: String(error) };
}

export function withErrorHandler<T>(
  handler: () => Promise<T>
): Promise<T | { code: string; message: string }> {
  try {
    return await handler();
  } catch (err) {
    return formatToolError(err);
  }
}
```

### Usage in Tool Handler

```typescript
// tools.ts
server.setRequestHandler(Tool, async (req: ToolRequest) => {
  return withErrorHandler(async () => {
    const args = InputSchema.parse(req.params.arguments);
    const result = await jiraClient.method(args);
    return formatOutput(result);
  });
});
```

## Input Validation with Zod

### Schema Pattern

```typescript
import { z } from 'zod';

// Define schema per tool
const ListIssuesSchema = z.object({
  project: z.string().optional(),
  maxResults: z.number().int().positive().default(10)
});

const LogWorkSchema = z.object({
  key: z.string().min(1, 'Issue key required'),
  hours: z.number().positive('Hours must be > 0').max(24),
  date: z.string().datetime().optional(),
  comment: z.string().optional()
});

// Type inference
type ListIssuesInput = z.infer<typeof ListIssuesSchema>;
type LogWorkInput = z.infer<typeof LogWorkSchema>;

// Validation in handler
async function handleListIssues(args: unknown) {
  const parsed = ListIssuesSchema.parse(args); // throws ZodError if invalid
  return await jiraClient.searchIssues(parsed.project);
}
```

## Async/Await & Promise Handling

### Pattern

```typescript
// Always use async/await (not .then)
async function handleRequest() {
  try {
    const issue = await jiraClient.getIssue(key);
    const formatted = formatIssueForAI(issue);
    return { content: [{ type: 'text', text: formatted }] };
  } catch (err) {
    return formatToolError(err);
  }
}

// No Promise chaining
// ❌ NOT: jiraClient.getIssue(key).then(...).catch(...)
// ✅ YES: await jiraClient.getIssue(key); try/catch
```

## Type Safety

### Avoid `any`

```typescript
// ❌ NOT
function processIssue(data: any) {
  return data.key; // Could be undefined
}

// ✅ YES
function processIssue(data: Issue) {
  return data.key; // Type-safe, guaranteed string
}

// ✅ If type unknown, use generics or narrowing
function processIssue<T extends { key: string }>(data: T) {
  return data.key;
}
```

### Optional Properties

```typescript
// Use ? for optional fields (not undefined unions)
interface Issue {
  key: string;              // Required
  description?: string;     // Optional (undefined allowed)
  customfield_10029?: number; // Story points (optional)
}

// Access with optional chaining
const storyPoints = issue.customfield_10029 ?? 0;
```

## Performance Considerations

### API Call Optimization

```typescript
// ❌ NOT: Sequential calls
const issues = await jiraClient.searchIssues(jql);
const details = [];
for (const issue of issues) {
  details.push(await jiraClient.getIssue(issue.key)); // N+1
}

// ✅ YES: Use JQL to fetch all fields at once
const issues = await jiraClient.searchIssues(
  jql,
  fields: ['key', 'summary', 'description', 'status', 'priority']
);
```

### Response Size

```typescript
// Limit maxResults to prevent memory bloat
const issues = await jiraClient.searchIssues(jql, { maxResults: 10 });

// Format only necessary fields for AI
function formatIssueListForAI(issues: Issue[]) {
  return issues.map(i => ({
    key: i.key,
    summary: i.fields.summary,
    status: i.fields.status.name,
    priority: i.fields.priority.name
  }));
}
```
